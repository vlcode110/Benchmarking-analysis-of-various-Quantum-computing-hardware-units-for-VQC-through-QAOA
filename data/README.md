# Data layout

All experimental outputs for the paper live here. Analysis code is in `../analysis/`; job submission code is in `../experiments/`.

## Folders

| Path | Contents |
|------|----------|
| `circuits/` | Canonical OpenQASM (`qvc_test_*.qasm`, `xeb_d01_c*.qasm`) and `weights_10q.npy` |
| `ideal/` | Classical statevector reference CSVs |
| `processed/` | Per-device summary CSVs and multi-device tables for the paper |
| `raw/` | Primary raw data — histogram JSON and XEB summaries |

## VQC raw data (`raw/`)

| Folder | Backend |
|--------|---------|
| `ibm_kingston/` | IBM Kingston |
| `ibm_fez/` | IBM Fez (VQC under `fez_vqc_*/`) |
| `ibm_marrakesh_vqc/counts/` | IBM Marrakesh |
| `braket_emerald/` | IQM Emerald |
| `braket_garnet/` | IQM Garnet |
| `braket_cepheus/` | Rigetti Cepheus |
| `braket_sv1/` | Braket SV1 simulator |
| `braket_tn1/` | Braket TN1 simulator |
| `ionq_native/20260707_084925Z/` | IonQ simulator + Forte-1 VQC (`*/vqc/counts/`) |

**Comparison subset:** circuit indices **0–10** only (`qvc_test_000` … `qvc_test_010`).

## XEB raw data

| Path | Backend |

| `raw/braket_tn1/xeb/` | Braket TN1 |

## Optional extension (not in main comparison notebook)

| Path | Notes |
|------|-------|
| `raw/ibm_marrakesh_xeb/` | IBM Marrakesh XEB depth sweep (120 circuits) |

## Processed tables (`processed/`)

- `vqc_*_summary.csv` — one row per (device, circuit)
- `vqc_multi_device_summary_long.csv` — combined VQC table
- `xeb_multi_device_*.csv` — XEB comparison tables

Generated by notebooks in `../analysis/`.
